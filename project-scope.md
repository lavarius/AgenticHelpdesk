## Problem
We receive hundreds of support emails daily. Our agents manually read, classify, and respond to each ticket - which is slow and leads to impmersonal, canned responses. Agents for a ticket management system are providing canned responses, and they are usually repetiive, we want to minimize repepetive work, as well as gives customers an impersonal response.

## Solution
Build a ticket management system that uses AI to automatically classify, respond to, and route support tickets - delivering faster, more personalized responses to students while freeing up agents for complex issues. The whole scope of this project is to create personalized messages to students within a reasonable time and have it take care of mundane tickets rather than the specialized ones.

## Ticket Statuses
- **Open** — newly created or awaiting agent response
- **Resolved** — issue has been addressed
- **Closed** — ticket is finalized and no further action needed

## Ticket Categories
- General Question
- Technical Question
- Refund Request

## User Roles
- **Admin** — the system is deployed with a default admin who can create and manage agents
- **Agent** — created by admin; handles tickets, reviews AI suggestions, and responds to students

## Features
- Receive support emails and create tickets
- Auto-generate human-friendly responses using a knowledge base
- Ticket list with filtering and sorting
- Ticket detail view
- AI-powered ticket classification (into one of the defined categories)
- AI summaries
- AI-suggested replies
- User management (admin only — create and manage agents)
- Dashboard to view and manage all tickets
